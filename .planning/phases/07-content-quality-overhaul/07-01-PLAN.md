Update the blog post page component to inject FAQPage JSON-LD schema when an FAQ section exists in the post content.

1. In `src/app/(main)/insights/[slug]/page.tsx`, after the existing Article JSON-LD:
   - Parse `post.content` (the raw MDX string from gray-matter) to detect FAQ section
   - Look for a heading like `## FAQ` or `## Frequently Asked Questions` followed by `### question` / answer pairs
   - Extract Q&A pairs using regex: match H3 headings as questions, and content between H3s as answers (strip markdown formatting for the schema text)
   - If FAQ pairs found (length > 0), build a FAQPage schema:
     ```json
     {
       "@context": "https://schema.org",
       "@type": "FAQPage",
       "mainEntity": [
         {
           "@type": "Question",
           "name": "question text",
           "acceptedAnswer": {
             "@type": "Answer",
             "text": "answer text"
           }
         }
       ]
     }
     ```
   - Render as a second `<script type="application/ld+json">` tag after the existing Article schema
   - Use a helper function `extractFAQs(content: string): Array<{question: string, answer: string}>` defined at top of file or in a utility

2. The extraction logic:
   - Find the index of `## FAQ` or `## Frequently Asked Questions` in the content
   - From that point, find all `### ` lines as questions
   - Content between consecutive `### ` lines (or end of content) is the answer
   - Strip markdown bold (**), links ([text](url) -> text), and trim whitespace
   - Limit to first 25 FAQs (matches the 20-25 target from CONTEXT.md)

3. Do NOT change any existing rendering, layout, or the Article JSON-LD schema. This is purely additive.

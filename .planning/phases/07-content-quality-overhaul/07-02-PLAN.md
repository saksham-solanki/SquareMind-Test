  <action>
Create `scripts/rewrite-prompt.md` -- the master prompt that the CLI script sends to Claude API for each post rewrite. This is the most critical artifact of the entire phase.

The prompt must instruct Claude to:

1. **Output format:** Pure MDX content (no frontmatter -- the script preserves and re-attaches the original frontmatter). Start directly with the TL;DR block.

2. **Structure (mandatory in this order):**
   - **TL;DR block** at the very top, before first H2. 3-5 bullet points summarizing key takeaways. Wrap in a blockquote (`> **TL;DR:** ...`)
   - **Introduction** (2-3 paragraphs, no H2 needed -- flows after TL;DR)
   - **Main sections** using H2 (`##`) for major topics and H3 (`###`) for subsections. No H4 or deeper. Aim for 6-10 H2 sections.
   - **Comparison table** (if the topic naturally supports it -- city comparisons, builder analysis, investment options, tax regimes). Use markdown table format.
   - **FAQ section** with heading `## Frequently Asked Questions`. Each question as `### Question text here?`. Answer as paragraph(s) below each H3. Target 20-25 questions. Mix of basic, intermediate, and advanced questions. Target "People Also Ask" long-tail queries.
   - **Sources section** with heading `## Sources`. Bullet list of 5-10 real external citations with URLs. Government portals (RERA state sites, income tax dept, SEBI), RBI data, NHB reports, reputable newspapers (Economic Times, Livemint, Moneycontrol). Use format: `- [Source Name](URL) — brief description of what this source provides`

3. **Content quality:**
   - Target 6000-8000 words. Some topics may be shorter -- that's OK, don't pad.
   - Voice: Advisory expert. SquareMind as trusted advisor. Use "We recommend...", "In our experience...", "Based on our analysis..."
   - Naturally reference SquareMind's experience -- weave in credibility signals. Link to consultation page where relevant.
   - Market data: Use ranges and trends ("₹7,000-9,000/sqft", "prices are rising") not exact figures.
   - Controversial topics: Present both sides but share SquareMind's assessment.

4. **Internal linking (10-15 links per post):**
   - The prompt receives a `LINK_MAP` variable with available internal links.
   - Use natural anchor text based on surrounding context, not fixed keyword strings.
   - For tool links, use the ToolCallout component format: `<ToolCallout tool="Tool Name" description="brief action description" url="/tools/slug" />`
   - Include 2-3 ToolCallout boxes per post where relevant tools exist.
   - Regular internal links use standard markdown: `[anchor text](/insights/slug)`
   - Do NOT link to the post itself. The script provides the current post's slug to exclude.

5. **What NOT to do:**
   - Do not include frontmatter (script handles this)
   - Do not use H1 headings (page title is rendered separately)
   - Do not use H4 or deeper nesting
   - Do not include a "Key Takeaways" section (TL;DR covers this)
   - Do not fabricate specific statistics or cite non-existent sources
   - Do not include images or image placeholders

The prompt should have template variables: `{{ORIGINAL_CONTENT}}`, `{{POST_TITLE}}`, `{{POST_CATEGORY}}`, `{{PRIMARY_KEYWORD}}`, `{{SECONDARY_KEYWORDS}}`, `{{CURRENT_SLUG}}`, `{{LINK_MAP}}`, `{{RELATED_TOOL_LINKS}}`

<task type="auto">
  <name>Task 2: CLI rewrite batch script</name>
  <files>scripts/rewrite-batch.js</files>
  <action>
Create `scripts/rewrite-batch.js` (plain Node.js, no TS) that rewrites MDX posts via the Claude API.

**CLI interface:**
```
node scripts/rewrite-batch.js --count 5
node scripts/rewrite-batch.js --count 10 --category "Investment Strategy"
node scripts/rewrite-batch.js --count 5 --dry-run
node scripts/rewrite-batch.js --status
```

**Arguments (use process.argv parsing, no external arg parser):**
- `--count N` (required for rewrite, default 5): Number of posts to process in this batch
- `--category "Name"` (optional): Only process posts in this category. If omitted, follows priority order: Investment Strategy -> Tax & Legal -> NRI Corner -> City Guides -> First-Time Buyers -> Builder Analysis -> Market Data -> Due Diligence -> Dark Truths
- `--dry-run` (optional): Show which posts would be processed without calling API
- `--status` (optional): Show rewrite progress by category (counts original vs rewritten posts by checking word count threshold)

**Core logic:**

1. **Load dependencies:**
   - `fs`, `path`, `child_process` (for git)
   - `gray-matter` for frontmatter parsing (already in package.json)
   - Load `content/link-map.json`
   - Load `scripts/rewrite-prompt.md`
   - Load ANTHROPIC_API_KEY from `.env.local` (parse manually: read file, find line starting with ANTHROPIC_API_KEY=, extract value)

2. **Select posts to rewrite:**
   - Read all MDX files from `content/posts/`
   - Determine "already rewritten" by word count: if post body > 3000 words, skip it (original posts are ~500 words)
   - Filter by --category if provided, otherwise use priority order
   - Take first --count posts that haven't been rewritten
   - Print selected posts and their categories

3. **For each selected post (sequential, not parallel -- rate limits):**
   - Read the MDX file, parse with gray-matter to separate frontmatter and content
   - Build the prompt by replacing template variables in rewrite-prompt.md:
     - `{{ORIGINAL_CONTENT}}` = original post body
     - `{{POST_TITLE}}` = frontmatter title
     - `{{POST_CATEGORY}}` = frontmatter category
     - `{{PRIMARY_KEYWORD}}` = frontmatter primaryKeyword
     - `{{SECONDARY_KEYWORDS}}` = frontmatter secondaryKeywords joined with ", "
     - `{{CURRENT_SLUG}}` = post slug (for self-link exclusion)
     - `{{LINK_MAP}}` = filtered subset of link-map.json (same category + tools + pages, exclude self). Limit to 50 most relevant entries to manage token count.
     - `{{RELATED_TOOL_LINKS}}` = tool entries from link-map.json formatted as a list
   - Call Claude API via fetch (POST to https://api.anthropic.com/v1/messages):
     - Model: `claude-sonnet-4-20250514`
     - Max tokens: 16000
     - System prompt: "You are a senior real estate content writer for SquareMind, an Indian real estate advisory firm. Follow the rewrite instructions exactly."
     - User message: the filled prompt template
     - Headers: `x-api-key`, `anthropic-version: 2023-06-01`, `content-type: application/json`
   - Extract the text response from `response.content[0].text`
   - Reconstruct the MDX file: original frontmatter (unchanged) + "---\n\n" + new content
   - Update readTime in frontmatter based on new word count (assume 200 words/min, round to nearest minute)
   - Write back to the same file path
   - Log: "Rewrote: {slug} ({old_words} -> {new_words} words)"
   - Wait 2 seconds between API calls (rate limit buffer)

4. **After batch completes:**
   - Print summary: X posts rewritten, average word count, any failures
   - If any failures, list them with error messages
   - Create git commit: `git add content/posts/ && git commit -m "content: rewrite batch — {count} {category} posts"`
   - Print remaining unrewritten posts count

5. **--status mode:**
   - Read all MDX files, count words in each
   - Group by category
   - Print table: Category | Total | Rewritten (>3000w) | Remaining | Avg Words
   - Print overall progress bar

6. **Error handling:**
   - If ANTHROPIC_API_KEY not found, exit with clear message
   - If API call fails, log error, skip that post, continue with next
   - If gray-matter parse fails, skip with warning
   - Wrap each post rewrite in try/catch so one failure doesn't stop the batch
  </action>

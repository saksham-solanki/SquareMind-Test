<task type="auto">
  <name>Task 2: Create mdx.ts utility and migrate all 10 posts to MDX files</name>
  <files>src/lib/mdx.ts, scripts/validate-blog.js, content/posts/*.mdx (10 files)</files>
  <action>
1. Create `src/lib/mdx.ts` with the following exports (matching the interface in 04-RESEARCH.md Pattern 1):
   - `PostMeta` interface: title, slug, description, category, tag, tags (string[]), publishedAt, readTime, views, primaryKeyword, secondaryKeywords (string[])
   - `Post` interface: { meta: PostMeta; slug: string; content: string }
   - `getAllPosts()`: reads content/posts/*.mdx via fs.readdirSync, parses each with gray-matter, returns Post[] sorted by publishedAt descending
   - `getPostBySlug(slug: string)`: reads single MDX file, returns Post | undefined
   - `getRelatedPosts(currentSlug, category, count=3)`: filters by same category, excludes current, returns first `count`

   IMPORTANT: The function signatures and return types must match what the existing blog pages expect. The existing `posts.ts` exports `Post` with fields: slug, tag, title, category, description, readTime, views, publishedAt, primaryKeyword, secondaryKeywords, content. The new `Post` type nests metadata under `meta` -- the blog pages will be updated in Plan 02 to use `post.meta.title` etc.

2. Migrate all 10 posts from `src/lib/posts.ts` to individual MDX files in `content/posts/`:
   For each post in posts.ts:
   - Create `content/posts/{slug}.mdx`
   - Extract YAML frontmatter from the post object fields: title, tag, category, description, readTime, views, publishedAt, primaryKeyword, secondaryKeywords
   - Convert the HTML content string to MDX-compatible markdown:
     * `<h2>text</h2>` -> `## text`
     * `<h3>text</h3>` -> `### text`
     * `<p>text</p>` -> plain paragraph (blank line separated)
     * `<strong>text</strong>` -> `**text**`
     * `<ul><li>` -> `- item` (markdown lists)
     * `<ol><li>` -> `1. item` (numbered lists)
     * `<a href="url">text</a>` -> `[text](url)`
     * HTML tables -> GFM markdown tables (| col | col |)
     * Remove all HTML tags, keep text content in markdown format
     * Use `---` for em dashes (already in some content as HTML entities)
   - Ensure no raw HTML remains that could cause JSX compilation issues (no unclosed tags, no class= attributes)

   The 10 slugs to migrate:
   - builders-delivery-timelines-india
   - real-estate-vs-mutual-funds-gold-india

3. Create `scripts/validate-blog.js` that:
   - Reads all .mdx files from content/posts/

   - Validates all 10 expected slugs exist
   - Prints pass/fail summary

4. Do NOT delete src/lib/posts.ts yet -- Plan 02 will update the blog pages to import from mdx.ts first, then remove posts.ts.
